# 功能优化：流式阶段输出

## 需求描述

用户点击"开始分析"后，在分析结果区域实时显示每个阶段的进度和结果，实现流式输出效果。

## 预期交互流程

### 阶段1：意图识别
```
[意图识别] 识别中...
[意图识别] ✓ 完成
意图识别结果：
- 分析类型：竞品分析
- 置信度：0.95
- 关键词：比亚迪、特斯拉、蔚来
```

### 阶段2：数据检索
```
[数据检索] 检索中...
[数据检索] ✓ 完成
数据检索结果：
- 向量检索：12条相关文档
- SQL查询：306个品牌，704个车型
- 总销量：13,260,185辆
```

### 阶段3：战略分析
```
[战略分析] 分析中...
[战略分析] ✓ 完成
PEST分析：
- 政治(P)：新能源购置税减免政策（正面）、双积分政策（中性）
- 经济(E)：GDP增长（正面）、消费信心（正面）
- 社会(S)：城镇化（正面）、环保意识（正面）
- 技术(T)：智能驾驶（正面）、800V高压（正面）

波特五力：
- 现有竞争者威胁：高（CR3=33.3%）
- 新进入者威胁：高（造车资质降低）
...
```

### 阶段4/5：报告生成
```
[报告生成] 生成中...
[报告生成] ✓ 完成
完整报告显示在下方...
```

## 实现方案

### 前端改造
1. 修改 `appendOutput()` 函数，支持追加内容而不是覆盖
2. 每个阶段完成后，调用 `appendOutput()` 追加该阶段的摘要
3. 新增 `clearOutput()` 清空之前的内容

### 后端改造
1. `progress_callback` 中传递该阶段的摘要信息
2. SSE 事件 data 中包含阶段摘要文本

### 数据格式建议
```json
{
  "event": "progress",
  "stage": "stage2",
  "stage_name": "数据检索",
  "status": "done",
  "summary": "向量检索：12条 | SQL查询：306品牌704车型"
}
```
