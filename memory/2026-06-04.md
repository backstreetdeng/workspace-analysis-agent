### 3. 提出架构重设计方案

#### 问题分析
```
❌ 当前错误架构：
用户 → workflow.py (Python流程控制) → SSE (流程控制) → 固定输出

问题：
1. workflow.py 是固定流程，AI无法动态决策
2. SSE 负责流式展示同时控制流程
3. Skills 调用被 Python wrapper 固定
```

```
✅ 正确架构（基于 agents-orchestrator）：
用户 → OpenProse Workflow (AI编排层) → SSE (仅流式展示)
                                    ↓
                              Skills (OpenClaw系统)
```

#### 架构原则
1. **AI 动态编排 > 固定流程控制**
2. **SSE 只做展示，不控制流程**
3. **Skills 通过 OpenClaw 系统调用**
4. **记忆持久化是基础**

### SKILL.md 核心结构

`
1. YAML frontmatter
   name: 技能名称（AI靠description判断何时激活）
   description: 【最关键】简洁描述能力

2. Body (Markdown)
   - 触发词/激活条件
   - 使用方法
   - 详细说明
   - 示例
`

### 关键原则

| 原则 | 说明 |
|------|------|
| description最重要 | AI靠它判断何时激活skill |
| 简洁性原则 | 只写AI不知道的 |
| 自由度匹配 | 高/中/低自由度选择 |
| 触发词明确 | 让AI知道何时使用 |
