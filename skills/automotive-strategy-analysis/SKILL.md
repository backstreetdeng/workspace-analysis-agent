# 乘用车市场战略分析 Skill

## 零、核心执行原则（必须遵守）

这个 Skill 不是“框架清单”，而是汽车市场战略分析智能体的大脑。Python 脚本、数据库、搜索工具只负责执行取数和保存文件，业务判断、证据选择、框架取舍、置信度评估必须由 LLM 按本 Skill 动态完成。

### 1. 证据先于结论
- 每个判断必须绑定证据来源、时间范围、指标口径和置信度。
- 没有证据时只能写“待验证假设”，不能写成确定结论。
- 用户问的是决策问题时，先找能影响决策的证据，不按固定模板堆内容。

### 2. 量化优先，定性兜底
- 能量化的内容必须给出具体数值、单位、时间戳和来源。
- 不能量化的内容必须标注“定性分析”，并说明依据和置信度。
- 所有评分必须说明评分逻辑，不能只给分数。

### 3. 分阶段交付中间产物
- 每个阶段都要形成可复查的中间产物：范围、假设、数据、证据账本、冲突记录、框架分析、最终报告。
- 最终报告必须能追溯到原始数据和证据账本。

### 4. 动态选择框架
- 不允许机械执行所有框架。
- 根据用户问题选择最小充分框架组合：市场机会优先市场规模+PEST+五力；品牌竞争优先竞品矩阵+SWOT+4P；政策影响优先PEST+情景分析；上市策略优先4P+竞品+用户需求。

### 5. 交叉验证和降级
- 核心数据至少 2 个独立来源交叉验证。
- 数据冲突时保留冲突，不强行平均；需要说明差异原因、口径差异和采用依据。
- 数据不足时降级输出低置信度结论，并列出补数路径。

## 触发词（激活条件）

当用户输入以下内容时自动激活：
- "分析XX品牌市场战略"
- "生成XX竞争对手SWOT"
- "评估新能源汽车市场吸引力"
- "做XX市场波特五力分析"
- "对比XX和XX的竞争优势"
- "乘用车市场战略"
- "新车上市战略分析"
- "品牌竞争格局分析"

## 五、标准执行工作流（7阶段）

### 阶段1：问题定义与范围界定
**目标**：把用户口语问题转成可分析、可验证、可交付的战略问题。

**输入**：用户原始问题、品牌/车型/市场/价位、地理范围、时间范围、目标输出形式。

**处理要求**：
- 识别问题类型：市场机会、竞品分析、政策影响、趋势判断、品牌战略、新车上市、投资评估、用户需求。
- 明确分析对象：品牌、车型、细分市场、价格带、能源形式、区域。
- 明确默认假设：用户未说明时，默认区域为中国乘用车市场，默认时间为最近12个月+未来3年预测，并在假设列表中标注。
- 判断是否需要追问：缺少品牌、地区、时间范围且会显著影响结论时，先追问；不显著时先按默认假设执行。

**输出**：
- `scope.md`：分析范围、问题类型、交付目标、排除范围。
- `assumptions.md`：关键假设、默认口径、潜在偏差。

**质量门禁**：
- 没有明确“分析对象”和“决策问题”不得进入数据采集。
- 所有默认值必须显式写入假设列表。

### 阶段2：分析路径设计
**目标**：先决定“为了回答问题，需要什么证据和框架”，再开始取数。

**处理要求**：
- 选择最小充分框架组合：
  - 市场机会：市场规模/TAM-SAM-SOM + PEST + 五力。
  - 品牌竞争：竞品矩阵 + SWOT+ + 4P。
  - 政策影响：PEST + 情景分析 + 风险矩阵。
  - 新车上市：4P + 竞品矩阵 + 用户需求/价格带分析。
  - 综合战略：市场规模 + PEST + 五力 + SWOT+ + 4P + 竞品矩阵。
- 为每个框架建立证据需求表，写清楚需要哪些指标、来源优先级、最低数据门槛。
- 明确可并行任务和串行依赖：数据采集可并行，结论综合必须在验证后串行执行。

**输出**：
- `analysis_plan.md`：框架选择理由、证据需求、并行/串行计划、降级方案。
- `evidence_requirements.csv`：结论点、所需指标、目标来源、最低置信度。

**质量门禁**：
- 每个预期结论必须至少映射到一个证据类型。
- 不允许因为 Skill 里有框架就全量执行。

### 阶段3：多源数据采集
**目标**：按证据需求获取结构化、可追溯的数据，而不是只抓文章片段。

| 数据类型 | P0来源 | P1来源 | P2来源 | 最低要求 |
|---------|--------|--------|--------|---------|
| 销量/份额 | 乘联会/中汽协 | 车企财报 | 盖世/媒体 | 数值+单位+月份 |
| 政策 | gov.cn/工信部/发改委 | 地方政府官网 | 权威媒体解读 | 原文链接+发布日期 |
| 价格/优惠 | 车企官网 | 汽车之家/懂车帝 | 终端调研 | 指导价/成交价口径分离 |
| 财务 | 年报/季报 | 交易所公告 | Wind/东方财富 | 币种+会计期间 |
| 用户反馈 | 车质网 | 汽车之家/懂车帝 | 社交平台 | 样本量+时间范围 |
| 技术/供应链 | 官方配置/专利 | 动力电池联盟/高工/佐思 | 行业新闻 | 指标口径+适用车型 |

**处理要求**：
- P0 数据源优先，P1 并行补充，P2 仅作降级或趋势补充。
- 每条数据必须记录：`metric`、`value`、`unit`、`period`、`source`、`url`、`retrieved_at`、`confidence`。
- 新闻和研报只能作为解释性证据，不能替代核心销量、价格、财务、政策原文。

**输出**：
- `raw_data.json`：原始数据。
- `sources.md`：来源清单。
- `evidence_ledger.csv`：证据账本。

**质量门禁**：
- 核心结论使用的数据必须有来源和时间戳。
- 结构化数据不足时，必须标记低置信度并触发补采或降级。

### 阶段4：数据验证、清洗与冲突处理
**目标**：先判断数据能不能用，再进入分析。

**处理要求**：
- 统一单位、币种、时间周期、车型口径和品牌归属。
- 对销量、份额、价格、财务等核心指标进行至少 2 个独立来源交叉验证。
- 识别冲突：来源口径不同、时间范围不同、统计对象不同、估算方法不同。
- 给每个关键指标打可靠性评分：1-5 分，并说明原因。

**输出**：
- `validated_data.json`：清洗后的指标。
- `conflicts.md`：冲突数据、采用/弃用理由。
- `data_quality.md`：数据新鲜度、完整性、可靠性评分。

**质量门禁**：
- 未通过验证的数据只能进入“背景参考”，不能支撑主要结论。
- 冲突未解释不得直接输出单一确定结论。

### 阶段5：专业框架分析
**目标**：用框架组织证据、暴露结构性矛盾、生成可比较的评分。

**处理要求**：
- 市场规模/TAM-SAM-SOM：给出总市场、可服务市场、可获得市场；计算历史增长和预测 CAGR。
- PEST：每个维度输出不超过 5 条洞察，每条绑定证据和影响方向。
- 波特五力：每力 1-5 分，说明权重、证据和置信区间。
- SWOT+：每项不超过 4 条，生成 TOWS 策略，并给出优先级分数。
- 4P：围绕产品、价格、渠道、促销给出与竞品相关的具体差异。
- 竞品矩阵：最多 5 个核心竞品，坐标轴必须是可解释指标，不能使用空泛标签。

**输出**：
- `framework_analysis.md`：各框架分析。
- `scores.json`：评分、权重、置信度。
- `competitor_matrix.csv`：竞品对比数据。

**质量门禁**：
- 框架分析必须区分事实、推断、建议。
- 不能只有框架结论，没有数据支撑。

### 阶段6：综合洞察与战略建议
**目标**：把多个框架收敛成少量真正能影响决策的洞察。

**处理要求**：
- 合并重复结论，识别各框架共同指向的关键矛盾。
- 给出优先级排序，建议使用：
  `优先级 = 影响程度 × 证据置信度 × 可执行性 × 紧迫性`
- 输出 3-5 条关键洞察，每条必须包含：结论、证据、影响、建议动作、风险。
- 必要时给出情景分析：乐观/基准/悲观。

**输出**：
- `insights.md`：关键洞察卡片。
- `recommendations.md`：行动建议、优先级、资源要求、风险。

**质量门禁**：
- 建议必须能回扣用户原始问题。
- 不能把 PEST、SWOT、4P 的摘要拼接当作最终洞察。

### 阶段7：报告生成与交付复核
**目标**：生成可交付、可追溯、可继续深挖的战略报告。

**报告结构**：
1. 执行摘要：3-5 条关键结论。
2. 问题与范围：对象、时间、地区、假设。
3. 方法与数据：框架、来源、数据质量。
4. 核心分析：按问题选择框架呈现，不机械堆全框架。
5. 综合洞察：事实、推断、建议分开。
6. 风险与不确定性：缺失数据、冲突、低置信度点。
7. 附录：证据账本、数据源、评分逻辑。

**输出文件**：

## 六、量化标准（强制要求）

### 数量限制
- PEST 每个维度：≤5条
- 波特五力 每力：≤3条核心数据
- SWOT 每项：≤4条
- 4P 每项：≤5条
- 竞品对比：核心竞品≤5个

### 质量要求
- 所有数值型数据必须包含：具体数值 + 单位 + 数据来源 + 时间戳
- 所有评分型数据必须包含：分数 + 置信区间（如 3.2±0.5）
- 所有趋势判断必须包含：历史数据 + 预测数据 + CAGR
- 定性分析必须标注：分析依据 + 置信度（高/中/低）

### 置信度标准
| 置信度 | 标准 |
|--------|------|
| 高 | 多源数据交叉验证，数据新鲜度<3个月 |
| 中 | 单源数据，数据新鲜度3-6个月 |
| 低 | 估算/推算，数据新鲜度>6个月 |

```bash
if [ ! -d "$TOPIC_DIR" ]; then
    echo "Error: Failed to create output directory"
    exit 1

```

### 2. 执行针对性数据提取策略

1. 先生成 `scope.md` 和 `assumptions.md`。
2. 再生成 `analysis_plan.md` 和 `evidence_requirements.csv`。
3. 按证据需求并行采集 P0/P1/P2 数据源。
4. 写入 `raw_data.json`、`sources.md`、`evidence_ledger.csv`。
5. 执行清洗、交叉验证和冲突记录。
6. 再进入框架分析和综合洞察。

| 分析类型 | P0 | P1 | P2 | 质量门槛 |
|---------|----|----|----|---------|
| 市场规模 | 中汽协/乘联会年报 | 盖世汽车/36氪 | 车企财报 | 至少2源或标低置信度 |
| 竞品分析 | 乘联会销量数据 | 汽车之家/懂车帝 | 财报/官网 | 指标口径一致 |
| PEST分析 | gov.cn/工信部 | 地方政府官网 | 行业媒体 | 政策以原文为准 |
| SWOT/五力 | 乘联会/中汽协 | 盖世/车质网 | 研报 | 评分有证据和权重 |
| 4P分析 | 财报/官网配置 | 汽车之家报价 | 终端调研 | 价格口径分离 |

**并行收集原则**：
- P0数据源优先获取（必须成功）
- P1数据源并行抓取（至少成功1个）
- P2数据源只做降级补充或趋势解释
- 核心数据不足时不得强行输出高置信度结论
